<objective>
Build the NexusMods authentication spine end-to-end: a user can log in (OAuth2+PKCE primary, API-key paste fallback), their long-lived credential is stored only in the system keyring, and a minimal account panel reflects logged-in / logged-out / Premium-Free / no-keyring states. This is the first vertical slice of Phase 3 — it stands up the headless `crates/nexus` crate (error + model + auth) and the shell's OS-integration (keyring + OAuth orchestration + commands + UI) that the download slices (Plans 02, 03) build on.

Purpose: Prove the auth + client + keyring spine works through the whole stack (headless client → shell keyring → IPC → UI) before any download flow is layered on. Honors the locked decisions: OAuth2+PKCE primary with API-key fallback (NEXUS-01), keyring-only storage with hard-fail-no-plaintext (NEXUS-02), headless `crates/nexus` with zero Tauri deps and OS-integration in the shell.

Output: New `nextwist-nexus` crate (lib/error/model/auth + mockito tests); `src-tauri` keyring + auth orchestration modules; `commands/nexus.rs` thin adapters; `AppState` carrying the async `NexusClient` + in-memory access token; account panel in the Svelte UI; new root workspace deps (oauth2, governor, futures-util, mockito) and shell deps (keyring 3.6, deep-link/single-instance pins for later plans).
</objective>
