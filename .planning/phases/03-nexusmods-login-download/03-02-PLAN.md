<objective>
Build the Premium download-into-staging slice end-to-end: a logged-in Premium user starts an in-app download of a Nexus mod, the file streams with a live per-item progress bar (no UI freeze), the client respects NexusMods rate limits proactively, and on completion the archive flows through the existing `extract`→staging pipeline to become an ordinary `ManagedMod` carrying persisted Nexus provenance — immediately deployable by the safe engine from Phases 1-2.

Purpose: Deliver the second vertical slice — the actual mod-acquisition value. It layers the HTTP client, rate limiter, and streaming download onto the Plan-01 auth spine, adds the V4 provenance migration + store facade + core DTO, and surfaces the downloads list UI. Honors the locked decisions: hybrid GraphQL-v2-metadata / REST-v1-download-link client (NEXUS-03/05), governor + X-RL-* backoff (NEXUS-05), reuse the extract→staging pipeline verbatim with persisted provenance (NEXUS-06), async streaming so the UI never freezes.

Output: `crates/nexus` client/ratelimit/download modules + mockito tests; `V4__nexus_provenance.sql` + `crates/store/src/nexus.rs` facade; `core::NexusSource` DTO; `commands/downloads.rs` adapter that wraps the headless progress callback into Tauri events and reuses `extract::install_archive`; downloads-list UI with per-item progress, states, and the rate-limit notice.
</objective>
