<objective>
Build the free-user nxm:// one-click handoff slice end-to-end: a user clicks "Mod Manager Download" on nexusmods.com, the OS routes the keyed nxm:// link to the running NexTwist instance (a second invocation is forwarded to the live app, never a duplicate window), NexTwist parses and redeems the free-user key+expires through the same download→extract→staging path proven in Plan 02, and a new downloads-list row plus a confirmation toast appear. This slice also closes the Plan-01 OAuth loop by routing the nxm://oauth/callback variant to the code-exchange.

Purpose: Deliver the third vertical slice — the free-user acquisition path and the live deep-link wiring that the whole phase's website integration depends on. It layers the Linux deep-link + single-instance plugins onto the Plan-01/02 spine and reuses the Plan-02 streaming-download + downloads-list verbatim for the redeemed file. Honors the locked decisions: nxm:// handoff for free users with key+expires redemption (NEXUS-04), deep-link + single-instance forwarding (NXM-01), single-instance registered FIRST, strict nxm:// parsing as a security boundary.

Output: headless `nexus::NxmLink` parser + tests; tauri.conf.json deep-link scheme + capability; single-instance-first + deep-link plugin registration + register_all() + on_open_url routing in lib.rs; free-user redemption wired through the Plan-02 start_download; the OAuth-callback route into Plan-01's complete_oauth; nxm:// arrival toast + free-user hint in the UI.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS / browser → app (nxm:// link) | An untrusted URL from the OS deep-link handler crosses into the app — the primary new attack surface |
| second app invocation → live instance (single-instance forward) | A forwarded argv/URL reaches the running process |
| free-user key+expires → NexusMods API | The opaque key/expires are redeemed at the download_link.json endpoint |
| nxm://oauth/callback → OAuth exchange | The authorization code arrives from the OS and is exchanged (Plan-01 loop closes here) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-12 | Tampering / Spoofing | malicious or spoofed nxm:// link | mitigate | Strict headless parse (scheme=nxm, numeric ids, opaque key/expires); NEVER shell out / execute / interpolate link content (V5); the downloaded file still routes through `extract`'s zip-slip defense. |
| T-03-13 | Spoofing | OAuth CSRF on nxm://oauth/callback | mitigate | `complete_oauth` validates `state == csrf` (Plan-01) before exchange; PKCE binds the code; malformed callback → Err(Auth). |
| T-03-14 | Denial of Service | duplicate-instance / link-flood | mitigate | single-instance forwarding produces one row per link (no duplicate windows); each download is still rate-limited by the Plan-02 governor limiter. |
| T-03-15 | Information Disclosure | key/expires/code in logs | mitigate | No `tracing::` call in the parser or on_open_url handler takes key/expires/code; redaction discipline (V7) grep-gated. |
| T-03-16 | Tampering | open-redirect via a redeemed CDN URI | mitigate | Redemption reuses the Plan-02 client (`redirect(Policy::none())`, HTTPS/rustls, error_for_status); only Nexus's own response URIs are followed. |
| T-03-SC | Tampering | deep-link/single-instance crate installs | mitigate | Both plugins were approved in the RESEARCH Package Legitimacy Audit (official tauri-apps/plugins-workspace); declared/pinned in Plan 01; cargo-deny re-runs at the wave merge. |
</threat_model>
