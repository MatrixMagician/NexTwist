<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted collection.json → serde_json parser | The collection archive's manifest is attacker-authorable untrusted JSON |
| collection source URLs → fetch decision | `direct`/`browse`/`manual` source URLs are off-Nexus and must never be auto-fetched (SSRF) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Tampering / SSRF | Off-Nexus direct/browse/manual URL auto-fetch | mitigate | classify direct/browse/manual as Manual and NEVER fetch (locked decision); only `nexus` and `bundle` sources are actionable; the resolver issues no off-Nexus request |
| T-04-09 | Spoofing | Stale/spoofed modRule reference manipulating load order | mitigate | rules only reorder rank within the user's own collection; a reference matching no resolved mod is ignored, not trusted (Pitfall 4) |
| T-04-10 | Information Disclosure | Resolve gate bypass (downloading before the report is accepted) | mitigate | resolve performs only metadata reads (zero downloads); a test asserts no download request is issued; the download path (Plan 04) is gated behind report acceptance |
| T-04-11 | Denial of Service | Unbounded collection.json (huge mods array) | accept | manifest is config-sized; serde_json bounds allocation; per-mod metadata reads are rate-limited via the shared governor |
| T-04-SC | Tampering | crates/cargo package installs in this plan | mitigate | No NEW package added (serde_json already a nexus dep; quick-xml audited in Plan 01); `cargo deny` runs at wave merge |
</threat_model>
