<objective>
Deliver the full Collection lifecycle end-to-end (COLL-02, COLL-03, COLL-04, COLL-05): a Premium user accepts the resolve report, NexTwist bulk-downloads the available mods, replays each mod's FOMOD choices headlessly, maps the Collection's rules onto the existing rank + load-order model, deploys the Collection as a dedicated profile so the modded game launches, and cleanly + reversibly uninstalls it leaving the game byte-for-byte pristine.

Purpose: This is the headline Collection capability. It adds ZERO new engine primitives — bulk download reuses `run_download_to_window` + the shared governor; deploy = `create_profile`→`set_profile_mod`→`switch_profile`; uninstall = `purge`→`delete_profile`→remove staged mods; pristine is asserted by the testkit blake3 harness. The only new logic is the headless `IChoices`→`fomod::Selection` replay and the thin orchestration adapter. Premium-only is enforced via `UserInfo.is_premium` (locked decision; no free-user fallback).

Output: `commands/collections.rs` (resolve/download/deploy/uninstall adapters) + registration; `nexus/replay.rs` (headless choice replay); the Collections UI per UI-SPEC §B/§C; a `collection_round_trip` pristine regression test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| collection manifest → bulk download orchestration | Pinned modId/fileId drive which files are downloaded |
| collection mods → deploy/purge engine | Downloaded mods cross into the safety-critical deploy/purge path |
| webview → collection command adapters | The IPC carries slug/revision + accept signals |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-12 | Tampering / SSRF | Auto-fetching off-Nexus (direct/browse/manual) collection mods | mitigate | the resolver (Plan 03) classifies them Manual; download_collection only iterates the Available nexus set + bundle files — off-Nexus URLs are never requested |
| T-04-13 | Tampering | A downloaded collection mod archive carrying zip-slip/symlink payloads | mitigate | every mod archive routes through `extract::install_archive` (zip-slip/symlink/`..` defense unchanged) + the Plan-01 root-detection; no new file primitive |
| T-04-14 | Tampering | Partial-deploy leaving orphaned files (uninstall not pristine) | mitigate | deploy = `switch_profile` (journaled, crash-safe), uninstall = `purge` (purge-to-pristine); `collection_round_trip` asserts byte-for-byte pristine via the testkit blake3 harness |
| T-04-15 | Spoofing | A pinned modId/fileId pointing at an attacker-swapped file (hash mismatch) | mitigate | the manifest carries per-file md5; integrity is a hint and the user accepts the resolve report; download routes through the rustls-only, error_for_status Phase-3 client; archived/unavailable files are surfaced, not silently substituted |
| T-04-16 | Elevation of Privilege | Premium gate bypass downloading on a free account | mitigate | `UserInfo.is_premium` checked before any download; non-Premium returns the notice and starts nothing (no nxm:// fallback this phase) |
| T-04-SC | Tampering | crates/cargo package installs in this plan | mitigate | No NEW package added (all deps audited in Plans 01/03); `cargo deny` runs at wave merge |
</threat_model>
