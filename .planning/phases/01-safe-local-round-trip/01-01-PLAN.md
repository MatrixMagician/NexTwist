<objective>
Scaffold the NexTwist Rust workspace and build the persistence + shared-types foundation that every later slice links against. This is the Walking Skeleton's spine-bearer: it installs the toolchain, defines the workspace, creates the headless `core` domain crate, the `store` crate (SQLite via rusqlite bundled + refinery migrations holding the game registry, the per-file deploy manifest, the write-ahead operation journal, and the content-addressed vanilla backup table), a `testkit` helper for byte-for-byte pristine assertions, and the cargo-deny policy that bans non-free UnRAR code.

Purpose: The deploy engine (Plan 04/05), the steam detector (Plan 02), and the extractor (Plan 03) all depend on `core` types and `store` persistence. Building these first as a shared, dependency-light foundation (per the locked multi-crate decision and ARCHITECTURE.md build order step 1) lets every later crate be unit/property-tested headless in CI with zero Tauri dependency.

Output: A compiling Cargo workspace with `crates/core`, `crates/store`, `crates/testkit`, a pinned `rust-toolchain.toml`, `deny.toml`, and a `.gitignore`. The op_journal + vanilla_backup + manifest schema established here is load-bearing for crash-safety (DEPLOY-06) and backup-before-overwrite (DEPLOY-04), so its shape is a contract for Plan 04/05.
</objective>

crates/core domain types (the shared vocabulary):
- `struct Game { appid: u32, name: String, install_dir: PathBuf, prefix: PathBuf, staging_dir: PathBuf }`
- `struct ManagedMod { id: i64, name: String, staging_root: PathBuf, enabled: bool }`

- `enum DeployMethod { Reflink, Hardlink, Symlink, Copy }`
- error enums via `thiserror` in libs (anyhow only at the app/Tauri boundary)

crates/store public surface (downstream callers use these — no SQL leaks outside store):
- `fn open(db_path: &Path) -> Result<Store>` — opens rusqlite Connection, sets `PRAGMA journal_mode=WAL`, `PRAGMA synchronous=FULL` for journal commits, runs refinery migrations
- registry: `add_managed_game(&Game)`, `list_managed_games() -> Vec<Game>`, `get_game(appid) -> Option<Game>`
- manifest: `record_deployed_file(&FileEntry)`, `list_deployed_files(appid) -> Vec<FileEntry>`, `remove_deployed_file(target_rel)`
- journal: `begin_op(intent) -> JournalId`, `mark_done(JournalId)`, `pending_ops() -> Vec<JournalRow>`
- vanilla: `backup_key_exists(hash) -> bool`, `record_vanilla(appid, target_rel, hash)`, `vanilla_for(appid, target_rel) -> Option<hash>`

| Boundary | Description |
|----------|-------------|
| dependency supply chain → built binary | Crates pulled from crates.io are linked into a binary intended for AppImage redistribution (DIST-02 pre-positioning) |
| filesystem ← store | The store writes a SQLite DB file under an app-data path; it must never write outside it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Cargo dependency graph (non-free/vulnerable dep slips in) | mitigate | `deny.toml` bans `unrar`/`unrar_sys` by name + license allow-list; `cargo deny check` is runnable now and wired into CI in Plan 06 |
| T-01-02 | Tampering | SQLite DB durability (crash corrupts journal/manifest) | mitigate | `PRAGMA journal_mode=WAL` + `synchronous=FULL` for journal-commit durability (RESEARCH.md A7); the op_journal/vanilla schema is the substrate for DEPLOY-06 recovery |
| T-01-03 | Information Disclosure | SQL injection via mod/game names | accept | All store writes use parameterized rusqlite `params!` bindings (no string-built SQL); no untrusted SQL surface |
| T-01-SC | Tampering | crates.io installs | mitigate | All 12 crates legitimacy-audited OK in RESEARCH.md §Package Legitimacy Audit (no [ASSUMED]/[SUS]/[SLOP]); versions pinned in `[workspace.dependencies]`; cargo-deny advisories check available |
