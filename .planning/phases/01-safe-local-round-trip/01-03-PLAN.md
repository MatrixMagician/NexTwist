<objective>
Build `crates/extract`: the pure transform from an untrusted local mod archive (`.zip` / `.7z` / `.rar`) into a validated, read-only per-mod staging tree. Every archive entry is path-validated (enclosed_name + re-canonicalize-under-root + reject symlink/absolute/`..` entries) before its bytes are written; extraction happens to a temp dir and is only moved into staging after the whole archive validates; `.rar` is handled exclusively via the system `unrar`/`7z` binary (argv, never a shell string) with a clear error when neither is present, so NO non-free RAR code is bundled.

Purpose: This is the "install a local mod archive into staging" segment of the Walking Skeleton spine. It is independent of steam detection (Plan 02) and the deploy engine — it depends only on `core` types and `store` from Plan 01 — so it runs in parallel with Plan 02. The zip-slip / malicious-symlink defense is the Phase 1 security centerpiece (untrusted third-party content is the entire threat surface this phase).

Output: `crates/extract` with a shared per-entry validator, format handlers (zip / sevenz-rust2 / system-rar), a staging pipeline, a crafted-malicious-archive rejection test, and a multi-format extract test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted mod archive → extractor | Third-party archive bytes cross into the filesystem; THE Phase 1 threat surface |
| archive path → system rar tool | A user/archive-controlled path string crosses into a spawned process |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Tampering / Elevation | zip-slip path traversal (../, absolute) | mitigate | `enclosed_name()` + re-canonicalize-under-root assertion in the shared validator; crafted-archive rejection test |
| T-01-08 | Tampering / Elevation | symlink-write-through in archive (CVE-2025-29787) | mitigate | Reject symlink entries explicitly; never create or follow them; zip >= 8.x (patched); validator covers zip + 7z + system-rar output |
| T-01-09 | Tampering | malicious archive overwrites files outside staging | mitigate | Extract to temp, validate the whole tree, then move into staging — never extract into staging/game tree directly |
| T-01-10 | Elevation | .rar via system tool command injection | mitigate | `Command::new(tool)` with archive path + output dir as separate argv elements (no shell interpolation); validate the path is an existing file first |
| T-01-SC | Tampering | crates.io installs (zip, sevenz-rust2) | mitigate | Legitimacy-audited OK in RESEARCH.md; zip >= 8 (CVE fixed in 2.3.0); sevenz-rust2 maintained fork; versions pinned; cargo-deny advisories check available |
</threat_model>
