<objective>
Close GAP-01 (blocker, DEPLOY-03): `purge()` removes deployed files and restores vanilla
backups but leaves behind the empty directories `deploy()` created, so after
install→deploy→purge the game `Data/` is NOT byte-for-byte pristine (real-world repro on a
live Skyrim SE install: 3 orphan empty dirs). The automated `round_trip_pristine` proptest
missed this because `testkit::snapshot_tree` hashes file contents only and is blind to
empty directories.

Fix in three coordinated moves, ordered RED→GREEN so the test proves the bug before the
fix lands:
1. Make the bug reproducible: strengthen `testkit` to be directory-aware and have
   `round_trip_pristine` (and the verify test) assert directory shape — this currently
   FAILS on orphan empty dirs.
2. Make it pristine: fix `purge()` AND the purge branch of recovery (`replay_purge`) to
   remove the now-empty directories `deploy()` created — a manifest-derived, bottom-up
   `remove_dir` cleanup bounded at the deploy root.
3. Extend `verify`/`repair` (DEPLOY-07) to detect (verify) and remove (repair) orphan empty
   directories under the deploy root.

Purpose: The non-destructive, fully-reversible round-trip is the product's #1 guarantee
(core value). "Pristine" must mean the directory tree too, not just file contents.
Output: A directory-aware pristine assertion + a manifest-derived empty-dir cleanup in
purge/recovery + verify/repair coverage of orphan empty dirs, all green on tmpfs and btrfs.
</objective>
