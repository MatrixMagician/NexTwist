<objective>
Wire the proven headless round-trip into a runnable application: a thin Tauri 2 shell with 3-10 line command adapters (zero business logic, per the locked decision) over `crates/steam`, `crates/extract`, and `crates/deploy`; a functional-minimal Svelte 5 SPA (game list with resolved paths, install-mod-from-archive, deploy/purge buttons, fs-warning surfacing); a startup `recover_on_launch` so any interrupted prior operation is recovered before the UI is served; and a CI workflow running `cargo test --workspace` + `cargo deny check`. This closes the Walking Skeleton: a real user can launch NexTwist, add a Bethesda game, install a local archive, deploy, and purge to pristine through the UI.

Purpose: Plans 01-05 built and tested the entire safety core headless. This plan makes it a usable desktop app (success criteria #1-#5 become user-reachable through the UI) without putting any safety logic in the webview layer. The human-verify checkpoint at the end confirms the full UI round-trip and (where a real Proton install exists) the deferred "mod loads in-game" manual UAT.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| webview (Svelte) → Tauri commands | Untrusted UI input (file paths, appids) crosses the IPC boundary into the command adapters |
| user-picked archive/folder path → command → headless crate | A user-supplied path string is handed to extract/steam for validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-19 | Tampering / Elevation | business logic leaking into #[tauri::command] (untestable safety code) | mitigate | Adapters are 3-10 lines delegating to headless crates; all validation/safety logic lives in (tested) crates, not the command layer (Anti-Pattern 4) |
| T-01-20 | Tampering | user-supplied archive/folder path used unvalidated | mitigate | Path validation happens in extract (per-entry) and steam (add_game_by_folder marker check) — the command just forwards the path; no path trust at the UI layer |
| T-01-21 | Tampering | CI bypass lets a non-free/vulnerable dep ship | mitigate | CI runs `cargo deny check` (bans unrar + advisories + licenses) and `cargo test --workspace` on every push — pre-positions DIST-02 |
| T-01-SC | Tampering | npm installs (SvelteKit/Tauri frontend deps) | mitigate | Frontend deps come from the create-tauri-app Svelte template (first-party Tauri/Svelte packages); pin via package-lock.json committed; no [ASSUMED]/[SUS] packages introduced |
</threat_model>
