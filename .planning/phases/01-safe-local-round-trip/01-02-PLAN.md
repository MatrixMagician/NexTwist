<objective>
Build `crates/steam`: the headless crate that quarantines all Steam/Proton-layout knowledge and hands the deploy engine already-resolved absolute paths. It auto-detects installed Steam games (native + Flatpak roots), offers a manual add-game-by-folder fallback, accepts only the two supported Bethesda AppIDs (Skyrim SE 489830, Fallout 4 377160), resolves each game's install directory, derives the Proton prefix (`compatdata/<appid>/pfx`) that steamlocate does NOT expose, and produces a per-game canonical `Data/` casing map that the deploy engine uses to normalize mixed-case mod paths under Wine (DEPLOY-08).

Purpose: This is the "detect a Bethesda Proton game" segment of the Walking Skeleton spine. It is independent of the extractor (Plan 03) and the deploy engine (Plan 04/05) — it only depends on `core` types and `store` registry from Plan 01 — so it runs in parallel with Plan 03. Quarantining Proton path knowledge here (per the Architectural Responsibility Map) keeps the deploy engine Proton-agnostic.

Output: `crates/steam` with discovery, resolution, and casing modules plus a `resolve_game` integration test driven by synthetic `libraryfolders.vdf` / `appmanifest_489830.acf` / `compatdata/` fixtures (real Steam not required in CI).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-supplied folder path → add_game_by_folder | Untrusted path string crosses into the resolver (manual-add fallback) |
| Steam VDF/ACF files → parser | Steam-owned config files are parsed; malformed content must not crash or escape |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Tampering / Elevation | add_game_by_folder path | mitigate | Validate the supplied path is an existing directory containing the expected Bethesda markers (Data/ + executable) before persisting it as a managed game (V5 input validation) |
| T-01-05 | Denial of Service | keyvalues-serde parse of attacker-substituted .acf/.vdf | accept | Files are Steam-owned, not third-party mod content; parse errors return a typed error (no panic); low-value, local-only target |
| T-01-06 | Spoofing | Wrong-prefix resolution silently writing nowhere | mitigate | Prefix derived from Steam's own layout + `$STEAM_COMPAT_DATA_PATH`; re-resolved each session; no hardcoded `steamuser`/native-HOME assumptions (Pitfall 6) |
</threat_model>
