# Quick Task 260623-m42 — Summary

Fixed the broken v1.0 release pipeline so a versioned, optimized AppImage with a changelog
is produced as a downloadable GitHub Release.

## Root cause
The `v1.0` tag triggered `release.yml`, which **failed** (run 28033312651) at the bundle step:
`tauri build --bundles appimage --locked` → `error: unexpected argument '--locked' found`.
`tauri build` only forwards args after `--` to cargo, so `--locked` must be `-- --locked`.
No GitHub Release / AppImage was ever produced.

## Changes (commit `fce5c6f`)
- **`.github/workflows/release.yml`**
  - `args: --bundles appimage --locked` → `--bundles appimage -- --locked` (the fix that unblocks the build).
  - Added an `id: changelog` step that extracts the top numbered `## [x.y.z]` section of
    `CHANGELOG.md` (skips `[Unreleased]` and trailing link-defs) into a multi-line `$GITHUB_OUTPUT`.
  - Added `releaseBody: ${{ steps.changelog.outputs.notes }}` so the Release shows the changelog.
- **`src-tauri/tauri.conf.json`** — version `0.1.0` → `1.0.0` (AppImage/Release now shows v1.0.0).
- **`CHANGELOG.md`** (new) — Keep a Changelog format with the `[1.0.0]` MVP entry.

## Notes
- The release `[profile.release]` was already optimized (`lto = "thin"`, `strip = true`) — no
  profile change needed; "optimized AppImage" = the release-profile build the workflow produces.
- The changelog step reads only the maintainer-controlled `CHANGELOG.md` (no untrusted
  `github.event.*` input) — no Actions injection surface.

## Verified
- Changelog extraction yields the clean 46-line `[1.0.0]` section (no `[Unreleased]`/link-def leak).
- `release.yml` is valid YAML; `tauri.conf.json` is valid JSON with `version: 1.0.0`.

## Outstanding (needs user sign-off — release action)
The fix is committed on `main` but **not yet pushed**, and the published `v1.0` tag still points
at the old (failing) commit. To actually produce the release, the `v1.0` tag must be moved to the
fixed commit and re-pushed (force-update). The original run produced no artifact, so re-pointing
is safe — surfaced to the user for go-ahead. Future releases should tag full semver `vX.Y.Z`.
